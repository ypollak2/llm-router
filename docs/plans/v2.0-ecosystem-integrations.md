# LLM Router v2.0 — Ecosystem Integrations Plan

> claw-code · OpenClaw · Agno

**Theme**: llm-router becomes the universal routing layer for the entire emerging Claude/AI agent ecosystem — not just Claude Code.

---

## Background: What Each System Is

### claw-code
Open-source Rust rewrite of the Claude Code agent harness (emerged March 2026 from the accidental npm source-map leak). Architecture is nearly identical to Claude Code:
- Same hook system (`UserPromptSubmit`, `PostToolUse`, `Stop`)
- Same MCP tool protocol
- Settings in `~/.claw-code/settings.json`
- OpenAI-compatible API client (`crates/api-client`)
- Plugin/skill model

**Key difference from Claude Code**: No subscription pressure (no Anthropic OAuth) — every call is a paid API call. This means llm-router's free-first routing (Ollama → Codex → paid) is *even more valuable* here.

### OpenClaw
Formerly Moltbot, renamed January 2026. Open-source Claude Code alternative with:
- Multi-provider support (Claude, GPT-4o, Gemini, DeepSeek, Ollama)
- A **Skills** marketplace (similar to Claude Code hooks but shareable packages)
- Hook pipeline with `BeforeToolCall` gating
- MCP support (SSE and stdio)
- No built-in budget awareness

**Key difference**: OpenClaw users pay API rates for everything. They have the most to gain from routing (potentially 70–85% cost reduction). The Skills ecosystem means we can distribute as a first-class package.

### Agno
Python multi-agent framework (39k+ stars, formerly PhiData). Production-grade:
- `Agent`, `Team`, `Workflow` primitives
- Full MCP bidirectional support — can both consume and expose MCP servers
- `AgentOS` turns agents into REST/MCP APIs
- Supports 20+ model providers via a clean `Model` interface
- Teams allow agents to collaborate; Workflows provide structured execution

**Key difference**: Agno is a *framework*, not an agent harness. Integration means becoming a routing layer inside Agno's model dispatch — not just a tool.

## Integration Layers

There are four layers of integration depth, each building on the previous:

```
Layer 4 │ Native SDK adapters  │  RouteredModel for Agno, RouteredTeam
Layer 3 │ Hook installers      │  claw-code + OpenClaw hook auto-install
Layer 2 │ Skill packages       │  OpenClaw skill marketplace package
Layer 1 │ MCP protocol         │  Works today — zero code needed
```

## Layer 2: Hook Integration (claw-code + OpenClaw)

### 2a. claw-code Hook Install

claw-code uses a `settings.json` hook format nearly identical to Claude Code's. The only differences are:

| | Claude Code | claw-code |
|---|---|---|
| Settings path | `~/.claude/settings.json` | `~/.claw-code/settings.json` |
| Hook event names | `UserPromptSubmit`, `Stop` etc. | Same (confirmed from Rust source) |
| Hook format | `{"hooks": {"UserPromptSubmit": [...]}}` | Same |
| Subscription pressure | Anthropic OAuth | N/A — no subscription |

**Plan:**
1. Extend `install_hooks.py` with a `install_claw_code()` function that:
   - Detects `~/.claw-code/settings.json`
   - Writes the same hooks with the `skip_subscription_hint` flag set (no CC pressure in claw-code)
   - Registers the MCP server in claw-code's MCP config (`~/.claw-code/mcp.json`)
2. Add `--claw-code` flag to `llm-router install`
3. Auto-detect during `llm-router install` if claw-code is present

**Routing value**: claw-code users pay API rates for every call. The free-first chain (Ollama → Codex → Gemini Flash) is the entire value proposition. Expected savings: same 70–85% as non-subscription Claude Code users.

**Hook modifications needed**:
- `auto-route.py`: already works (no CC-specific logic in classifier)
- `session-end.py`: remove subscription pressure section (no `usage.json` for claw-code)
- `status-bar.py`: remove `CC N%s` prefix, show only `sub:0 · free:N · paid:N · $X saved`

### 2b. OpenClaw Skill Package

OpenClaw's Skills are installable packages. Creating `llm-router-skill`:

```
llm-router-skill/
├── skill.json          # name, version, description, hooks
├── hooks/
│   ├── before-prompt.py   # maps to UserPromptSubmit
│   └── session-end.py     # maps to Stop/SessionEnd
└── mcp.json            # registers llm-router MCP server
```

`skill.json`:
```json
{
  "name": "llm-router",
  "version": "1.6.0",
  "description": "Smart model routing — save 70% on API costs",
  "author": "ypollak2",
  "hooks": {
    "before-prompt": "hooks/before-prompt.py",
    "session-end": "hooks/session-end.py"
  },
  "mcp": "mcp.json"
}
```

OpenClaw skill install (after publishing to OpenClaw's skill registry):
```bash
openclaw skill add llm-router
```

**Distribution**: Submit to the OpenClaw Skills marketplace. This is a high-leverage distribution channel — OpenClaw has a large user base paying API costs and actively looking for cost reduction tools.

## Layer 3: Agno Native Integration (Highest Impact)

This is the most architecturally interesting integration and has the largest addressable user base.

### 3a. `RouteredModel` — Drop-in Replacement

Agno's model interface is a clean Python ABC. Every agent takes a `model=` parameter. The goal: make llm-router a transparent routing layer.

**Today (Agno agent, no routing):**
```python
from agno.agent import Agent
from agno.models.anthropic import Claude

agent = Agent(
    model=Claude(id="claude-sonnet-4-6"),
    instructions="You are a research assistant",
)
```

**With llm-router (proposed):**
```python
from agno.agent import Agent
from llm_router.adapters.agno import RouteredModel

agent = Agent(
    model=RouteredModel(),   # drop-in replacement — picks model per prompt
    instructions="You are a research assistant",
)
```

`RouteredModel` intercepts each `response()` / `stream()` call, runs the classifier on the prompt, then delegates to the appropriate Agno model provider.

**Implementation sketch** (`src/llm_router/adapters/agno.py`):
```python
class RouteredModel:
    """Agno-compatible model that routes each prompt through llm-router."""

    def __init__(
        self,
        profile: str = "balanced",
        budget_models: dict | None = None,
    ):
        self._profile = profile
        # Lazy-load actual model instances
        self._models: dict[str, Any] = {}

    async def response(self, messages, **kwargs):
        task_type, complexity = await classify(messages[-1].content)
        model = await select_model(task_type, complexity, self._profile)
        agno_model = self._get_agno_model(model)
        return await agno_model.response(messages, **kwargs)

    def _get_agno_model(self, model_id: str):
        """Cache and return the right Agno model instance."""
        if model_id not in self._models:
            self._models[model_id] = _build_agno_model(model_id)
        return self._models[model_id]
```

### 3b. `RouteredTeam` — Budget-Aware Multi-Agent

Agno Teams assign specialized agents to subtasks. The router can optimize the model for each agent's role:

```python
from agno.team import Team
from llm_router.adapters.agno import RouteredTeam

team = RouteredTeam(
    agents=[
        researcher,   # → llm-router routes to Perplexity/Gemini Pro
        analyst,      # → llm-router routes to Sonnet/GPT-4o
        writer,       # → llm-router routes to Haiku/Gemini Flash
    ],
    monthly_budget_usd=20.0,   # llm-router enforces this across all agents
)
```

Key benefit: in a 3-agent Team without routing, all agents default to the same expensive model. With `RouteredTeam`, each agent gets the cheapest model that fits its task complexity — the writer agent uses Haiku (90% cheaper), the researcher gets Perplexity, and only the analyst uses Sonnet.

### 3c. Agno MCP Server Consumption

Agno already supports consuming MCP servers. The 33 llm-router tools become Agno tools with two lines:

```python
from agno.tools.mcp import MCPTools

agent = Agent(
    model=Claude(id="claude-sonnet-4-6"),
    tools=[MCPTools(command="llm-router")],
)
```

This is Layer 1 but exposed as the idiomatic Agno pattern. Document it explicitly in the README and Agno's cookbook.

### 3d. Agno MCP Server Mode (reverse direction)

When Agno runs in `AgentOS` mode with `enable_mcp_server=True`, it exposes its agents at `/mcp`. This means:

- A Claude Code user can call an Agno multi-agent team as an MCP tool: `llm_orchestrate` could delegate to an Agno team
- llm-router becomes the routing gateway in front of Agno workflows
- The architecture: `Claude Code → llm_orchestrate → Agno Team (via MCP) → individual agents (via RouteredModel)`
